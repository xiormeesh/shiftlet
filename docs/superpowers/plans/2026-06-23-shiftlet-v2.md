# Shiftlet v2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Update shiftlet to support multi-cluster scenarios (same-host and cross-host) for testing the multicluster agentic OLS architecture.

**Architecture:** Shell scripts wrapping libvirt/KVM and the OpenShift agent-based installer. `common.sh` is the engine containing all logic; thin wrapper scripts (`create.sh`, `delete.sh`, etc.) are entry points. Env files (`.env`) serve as editable cluster profiles.

**Tech Stack:** Bash, libvirt/virsh, virt-install, iptables, OpenShift agent-based installer

## Global Constraints

- All logic lives in `common.sh`. Wrapper scripts are one-liners that source it.
- Privileged operations use `sudo` (user runs scripts from their terminal).
- Env files are gitignored (`*.env` in `.gitignore`).
- State lives at `/var/lib/shiftlet/`, install-time assets at `/tmp/shiftlet-<name>/`.
- iptables rules are tagged with comments (`shiftlet-<name>`, `shiftlet-interbridge`) for idempotent removal.

### Task 4: Add `sync_inter_bridge_rules()` and wire it into create/delete

Add the inter-bridge forwarding function and call it after cluster creation and deletion so same-host clusters can reach each other.

**Files:**
- Modify: `common.sh` — add new function, modify `create_cluster()` and `delete_cluster()`

**Interfaces:**
- Produces: `sync_inter_bridge_rules()` — called automatically, no external interface

- [ ] **Step 1: Add `sync_inter_bridge_rules()` function**

In `common.sh`, after the `remove_fw_rules()` function (after line 131), add:

```bash
sync_inter_bridge_rules() {
    if [[ ! -f "$SLOTS_FILE" ]] || [[ ! -s "$SLOTS_FILE" ]]; then
        return
    fi

    # Remove all existing inter-bridge rules
    local tmp
    tmp=$(mktemp)
    sudo iptables-save | grep -v "shiftlet-interbridge" > "$tmp"
    sudo iptables-restore < "$tmp"
    rm -f "$tmp"

    # Collect active bridges
    local bridges=()
    while IFS='=' read -r slot name; do
        [[ -n "$slot" && -n "$name" ]] || continue
        bridges+=("$(bridge_for "$slot")")
    done < "$SLOTS_FILE"

    # If 2+ clusters, enable ip_forward and add pairwise ACCEPT rules
    if [[ ${#bridges[@]} -ge 2 ]]; then
        info "Enabling inter-cluster routing for ${#bridges[@]} clusters"
        echo "net.ipv4.ip_forward=1" | sudo tee /etc/sysctl.d/99-shiftlet.conf >/dev/null
        sudo sysctl -w net.ipv4.ip_forward=1 >/dev/null
        for (( i=0; i<${#bridges[@]}; i++ )); do
            for (( j=i+1; j<${#bridges[@]}; j++ )); do
                sudo iptables -I FORWARD \
                    -i "${bridges[$i]}" -o "${bridges[$j]}" \
                    -m comment --comment "shiftlet-interbridge" -j ACCEPT
                sudo iptables -I FORWARD \
                    -i "${bridges[$j]}" -o "${bridges[$i]}" \
                    -m comment --comment "shiftlet-interbridge" -j ACCEPT
            done
        done
    fi
}
```
